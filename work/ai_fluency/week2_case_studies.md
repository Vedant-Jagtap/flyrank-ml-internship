# Case Study 1 - FlyRank Machine Learning Internship

## Problem

I wanted to understand how Machine Learning can be applied to real-world search data instead of only learning theory. The goal was to frame a business problem into an ML task and understand how data supports decision-making.

## What I Did

I completed the starter notebooks, explored the dataset, analyzed webpage-level features, and framed the project as a classification problem. I identified the target variable, selected evaluation metrics, and documented why Machine Learning performs better than fixed rules for this task.

## Outcome

I created a reproducible ML workflow, documented my research question, and prepared the foundation for my capstone project. The project demonstrates my ability to understand an ML problem before building a model.

# Case Study 2 - DocuMind AI

## Problem

Users struggle to find information inside large PDF documents quickly.

## What I Did

I built an AI assistant using FastAPI, LangChain, ChromaDB, and the Gemini API. The system processes uploaded PDFs, creates embeddings, stores them in a vector database, and answers questions using Retrieval-Augmented Generation (RAG).

## Outcome

The application enables users to interact with documents through natural language instead of manually searching hundreds of pages.

# Case Study 3 - Healthcare Readmission Prediction

## Problem

Hospitals need to identify patients who are likely to be readmitted so preventive actions can be taken.

## What I Did

I cleaned healthcare datasets, performed feature engineering, trained multiple ML models, evaluated them with classification metrics, and exposed predictions using FastAPI.

## Outcome

The project demonstrates an end-to-end Machine Learning workflow from data preprocessing to model deployment.
